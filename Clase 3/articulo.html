15 mejores prácticas de HTML que los desarrolladores deben conocer

Este post está dedicado a aquellos que acaban de entrar en el mundo del desarrollo web. A continuación encontrarás 15 consejos para escribir HTML.
Básicamente, esta es la guía para principiantes sobre las mejores prácticas de HTML. La mayoría de las páginas web utilizan HTML. Las prácticas
mencionadas a continuación no sólo le ayudarán a crear un código limpio y bien estructurado, sino que facilitarán la optimización de los motores de
búsqueda. Espero que esto ayude a llevar a cabo una conversión efectiva de PSD a HTML.

Cerrar las etiquetas

Aunque, es muy simple, pero aún así es una práctica común que vemos en la vida cotidiana. A veces, incluso omitimos las etiquetas < ol > y < ul >.
Además, muchos optaron por omitir las etiquetas LI de cierre también. Es urgente abordar esta cuestión, de lo contrario se enfrentará a problemas de
validación y compatibilidad en lo que respecta a la codificación.

Uso de etiquetas de título significativas

Una de las cosas más importantes que ayuda en el proceso de optimización de motores de búsqueda del sitio web es el uso de la etiqueta del título. La
etiqueta del título ayuda a que la página web sea significativa y amigable para los motores de búsqueda. El texto escrito en la etiqueta del título se
muestra en los SERPs.

Etiquetas meta descriptivas

Aparte de las etiquetas de título, otra cosa importante que ayuda a la optimización del motor de búsqueda del sitio web es el uso de meta etiquetas
descriptivas. Cuando un internauta teclea una palabra clave en la barra del motor de búsqueda, las etiquetas meta son los atributos que hacen que la
página web sea más significativa y, por lo tanto, las arañas del motor de búsqueda puedan localizar fácilmente la página deseada. ¿Cómo funciona?
La meta descripción indica de qué trata la página web. Describe brevemente el propósito del sitio web. La descripción difiere de una página a otra.
Debe escribir un texto preciso diferente en la etiqueta de meta descripción para cada página web.

Uso del metaatributo de palabras clave

Este atributo consiste básicamente en una lista de palabras y frases clave relacionadas y relevantes separadas por comas. Estas palabras clave también
favorecen que la página web tenga más sentido. ¡No seas malvado, no hagas spam! Por favor, no trate de hacer sólo el relleno de palabras clave, ya que
no va a ayudarle en el largo camino. Sólo tiene que añadir algunas palabras y frases que describan su página web de forma adecuada.

Evitar el uso de estilos en línea

Hay tres maneras de insertar una hoja de estilo. Un estilo en línea pierde muchas de las ventajas de las hojas de estilo al mezclar el contenido con la
presentación. Utilice este método con moderación. Para utilizar los estilos en línea, utilice el atributo style en la etiqueta correspondiente. El atributo
style puede contener cualquier propiedad CSS. El ejemplo muestra cómo cambiar el color y el margen izquierdo de un párrafo:

< p style="color:siena;margin-left:20px"> Esto es un párrafo.< / p >

Aunque se ve bien, pero puede causar errores en las prácticas de codificación. Cree siempre un archivo CSS separado para sus estilos. Evite hacer
estilos durante la codificación. Pero ya, una vez que haya terminado con la parte de codificación, puede comenzar con la adición de estilos.

Colocar los archivos JavaScript en la parte inferior

Es necesario colocar correctamente un JavaScript en una página HTML para asegurarse de que el navegador lo ejecute. No es especialmente difícil,
pero hay que separar el comportamiento (JavaScript) y la estructura (XHTML). Uno de los principales parámetros del algoritmo de Google para
clasificar un sitio web es el tiempo de carga del mismo. Uno de los principales objetivos del desarrollo web es reducir el tiempo de carga del sitio
web. Sólo hay que colocar esos archivos JavaScript (si el propósito es sólo añadir funcionalidad) en la parte inferior, justo antes de la etiqueta de
cierre del cuerpo porque cuando se carga un script, el navegador no puede continuar hasta que se haya cargado todo el archivo. Los usuarios tendrán
que esperar mucho tiempo antes de que aparezca algo en la pantalla.

Divida su diseño usando Divs

El uso de la etiqueta div le permite definir la página en términos de las divisiones lógicas de la página, no sólo las ubicaciones en el diseño. Si utilizas
la etiqueta div en combinación con CSS, crearás una página fácil de gestionar. La mayoría de las veces, una página web está constituida por diferentes
secciones de contenido, por lo que utilizando estas etiquetas para dividirlas, estarás preparado para darles el estilo que quieras.El uso excesivo de las
etiquetas div también es una mala práctica. Algunos desarrolladores utilizan esta etiqueta para envolver múltiples etiquetas div que contienen más
etiquetas div, creando una montaña de divs. No tiene sentido utilizar esta etiqueta como último recurso cuando no se encuentra ninguna otra etiqueta
adecuada.

Nombres de etiquetas en minúsculas

Mantener las marcas en minúsculas es una buena práctica. Aunque ponerlo en mayúsculas no causa ningún daño, afecta a la legibilidad del marcado.
Por lo tanto, es bastante bueno si mantiene todos los nombres de las etiquetas en minúsculas. También se ve bien a nuestros ojos.

Añadir el atributo alt a las imágenes

La etiqueta img permite insertar una imagen en una página HTML. Tiene dos atributos obligatorios: src y alt. El texto ALT debe componerse como una
alternativa textual adecuada a la imagen: a veces puede resultar en una descripción de la imagen, pero en la práctica esa elección parece ser errónea más
a menudo que correcta. Utilizar un atributo alt significativo es bastante importante para escribir un código válido y semántico. Puede que algunos hayan
ignorado esta parte, pero los desarrolladores necesitan rellenar estas secciones por razones de validación y accesibilidad.

Uso adecuado de los encabezados

Utilice las etiquetas adecuadas para mejorar el SEO de su sitio. Las etiquetas de los encabezados son un elemento muy importante para crear y mantener
un sitio web o un blog que funcione bien en los motores de búsqueda. Hay seis niveles de encabezados. Estos niveles son los siguientes H1, H2, H3, H4,
H5, H6. La etiqueta de cabecera H1 es la de mayor tamaño de letra y la H6 es la más pequeña. Los motores de búsqueda buscan las etiquetas de cabecera
porque forman representaciones para encontrar el contenido, así como las vías de acceso en su blog o sitio web.

Terminar la navegación con una lista desordenada

Casi todos los sitios web/blogs tienen algún tipo de estado de navegación. No es necesario utilizar la etiqueta div para envolver la sección de navegación.
En su lugar, se utiliza ul para terminar. La etiqueta ul está destinada a contener una lista de elementos.

Utilizar el restablecimiento de CSS

Es un tema bastante discutible sobre el uso del restablecimiento de CSS. La mejor salida posible es desarrollar nuestro propio restablecimiento de CSS.
Puedes empezar usando algún restablecimiento CSS popular como el de Eric Meyer, y luego gradualmente puedes incrustar tu propio estilo de acuerdo
a tus requerimientos. Si no haces esto, no entenderás realmente por qué tus elementos de la lista están recibiendo ese extra de relleno cuando no lo has
especificado en ninguna parte de tu archivo CSS.

Aprende a utilizar los elementos HTML correctos en el lugar correcto

Con el tiempo, asegúrese de conocer todos los elementos HTML. Utilízalos correctamente para conseguir una estructura de contenidos semántica y
con sentido.

Por ejemplo:

- El elemento p define un párrafo en el documento HTML.

- El elemento body define el cuerpo del documento HTML.

- ul (listas desordenadas) cuando el orden de los elementos de la lista no es importante.

- ol (listas ordenadas) cuando el orden de los elementos de la lista es importante.

- dl (listas de definición) para los pares artículo/definición.

- En lugar de utilizar i y b, utilice em para el énfasis y strong para el énfasis fuerte.

- En lugar de usar br para insertar una línea entre párrafos, utiliza las propiedades de margen y/o relleno de CSS.

Todas estas cosas juegan un papel importante en el éxito de la conversión de PSD a XHTML.

Ver la fuente

Nadie llega a ser perfecto en el primer movimiento. En las etapas iniciales, puede visitar algunos buenos sitios web y comprobar cómo han codificado
las diferentes secciones del sitio web. Incluso puedes buscar algún buen Javascript utilizado por otros y puedes aplicar lo mismo en tu trabajo. Mira la
fuente y busca en la etiqueta HEAD el nombre del script. Es bueno aprender de los demás, pero no sigas la mala práctica de robar el diseño de otros.

Validación de la codificación

Es necesario hacer la validación de su marcado continuamente. No es así. No es que una marca que no esté completamente validada sea una mala marca,
y que esté completamente validada sea un gran trabajo.

Pero aún así necesitas descargar la validación de marcas del W3C o la barra de herramientas para desarrolladores de Firefox con el propósito de validar.
Buscará errores, si los hay. Realice la tarea de validación con frecuencia, para que pueda evitar varios problemas que son difíciles de resolver de otra
manera una vez que haya terminado con el trabajo del proyecto.

Usted necesita trabajar en las prácticas de HTML anteriores con el fin de convertirse en un buen codificador. Básicamente, se trata de una guía para
principiantes, ya que los expertos podrían no pasar por alto estas buenas prácticas, pero la probabilidad de que los principiantes pasen por alto
alguno de estos aspectos es bastante alta. Así que, APRENDE, APRENDE y APRENDE.
